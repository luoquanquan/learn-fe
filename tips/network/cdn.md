# CDN

## 概念

CDN(Content Delivery Network), 内容分发网络。是根据用户的位置分配最近的服务器资源的技术。用户在访问资源是不再是必须访问源站，而是访问离他最近的 "边缘节点"请求响应的过程如下图：

## 原来分析

> 常规情况下，我们访问一个 url 时的流程主要为：
> 用于输入 域名 -> 浏览器携带域名访问 DNS 服务器 -> DNS 解析到目的主机 IP 地址 -> 根据 IP 地址请求目标服务器 -> 得到目标服务器返回

应用 CDN 之后，DNS 返回的不再是一个具体的 IP 地址，而是一个 CNAME 指向 CDN 全局负载均衡。根据之前提到的 DNS 域名解析的流程。由于没有返回 ip 地址。本地 DNS 服务器会再次向 CDN 全局负载均衡发送请求。分配权限转交给了 CDN 全局负载均衡。调度的过程如下：

1. 查看用户的 ip 地址，推断用户的地理位置并查找最近的边缘节点
2. 查看用户所在的运营商网络，找到相同网络的边缘节点
3. 检查边缘节点的负载情况，选择其中负载较轻的节点
4. 还会考虑边缘节点其他情况，诸如”健康情况”，服务能力，带宽，响应时间等等…

结合上边的因素，可以得到最合适的边缘节点。然后把这个节点的 IP 返回给本地 DNS 服务器并由其返回给用户浏览器

![](http://handle-note-img.niubishanshan.top/2021-12-09-20-39-50.png)

本地 DNS 访问 CDN 调度中心的设计简直太秀了，这样既保证了本次完美访问，本地有人下次访问时也能直接读取到最优的边缘节点，因为 DNS 服务器代为缓存了 6666

## 缓存代理

缓存系统也是 CDN 的一个关键的部分。分为两个层次，一级缓存和二级缓存。一级缓存直接连源站，二级缓存直连用户。CDN 的回源主要是指二级缓存中没有用户请求的资源而像一级缓存请求，如果一级缓存中也没有该资源才会向真正的源站请求。两个衡量 CDN 质量的指标：

### 命中率

用户访问的资源恰好在缓存系统里，可以直接返回给用户，命中次数与所有访问次数之比

### 回源率

缓存里没有，必须用代理的方式回源站取，回源次数与所有访问次数之比
