# http vs https

## 什么是 http / https

http(HyperText Transfer Protocol), 超文本传输协议, 是实现网络通信的一种规范.
作为一种传输协议, 即将数据由 A 传递给 B, 再由 B 传递给 A. A / B 之间可能存在很多第三方代理: A<=>X<=>Y<=>Z<=>B. 传输的数据也不是计算机底层的二进制, 而是完整的, 有意义的数据. 如 html css js 图片等超文本. 能够被上层应用识别

## http 的特点

1. 支持客户 / 服务模式
2. 简单快速: 客户向服务发送请求时, 只需要请求方法和路径. 由于协议简单使得 HTTP 服务程序规模小, 进而通信速度快.
3. 灵活: http 允许传输任何类型的数据对象, 通过 Content-Type 进行标记
4. 无状态: http 协议无法根据之前的状态对本次请求进行处理

## https 概念

由于 http 传递信息的过程中以明文的方式传输, 存在泄露的风险. 于是基于 http + ssl/tls 的 https 诞生了.

![](http://handle-note-img.niubishanshan.top/2021-12-09-18-54-04.png)

1. 首先客户端通过 url 向服务端请求建立 https 链接
2. 服务端收到请求后会将其支持的证书信息(包含公钥)传递给客户端
3. 客户端和服务器协商 ssl 链接的安全等级, 也就是信息加密的等级
4. 客户端根据双方协商的安全等级建立会话密钥, 然后利用服务器返回的公钥对密钥进行加密并上传给服务器
5. 服务器利用自己的私钥解密出会话密钥
6. 两端利用会话密钥加密彼此间的通信内容, 由于中间人无法得知两者约定的密钥, 所以无法获知两者之间传递的内容

## http 和 https 的区别

1. https 是 http 协议的安全版本, 相对于 http 的明文传递 http 添加了 ssl/tls 加密. 相对更安全
2. 两者默认端口也不一样 http: 80 https: 443
3. Https 通信的过程中设计到加密以及握手, 性能略输于 http
4. https 需要 ssl, ssl 证书需要花钱, 功能越强的证书往往越贵
