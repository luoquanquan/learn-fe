**考点**: 响应式系统 / 框架原理 / Signal

**答案要点**:

## 3. Transformer 架构中的 Self-Attention 机制是如何工作的？为什么它比 RNN 更适合处理长文本？

**考点**: Transformer / Attention / 大模型架构

**答案要点**:

**Self-Attention 公式**:
```
Attention(Q, K, V) = softmax(QK^T / √d_k) · V
```

**计算过程**:
1. 将输入转换为 Q、K、V 矩阵
2. 计算注意力分数：Q × K^T
3. 缩放、Softmax 归一化
4. 加权求和：× V

**为什么比 RNN 强**:
| 特性 | RNN | Transformer |
|------|-----|-------------|
| 依赖距离 | 长距离衰减 | O(1) 直接连接 |
| 并行度 | 串行 | 全并行 |
| 训练速度 | 慢 | 快（GPU 友好）|

## 4. 什么是 Tree Shaking？Webpack、Rollup、Vite 在 Tree Shaking 实现上有什么区别？

**考点**: 构建工具 / 模块系统 / 代码优化

**答案要点**:

**Tree Shaking 原理**:
- 静态分析 ESM 的导出/导入
- 标记未使用代码（DCE - Dead Code Elimination）
- 依赖 ES Module 的静态结构

**工具对比**:
- Webpack: 需配置 optimization.usedExports，兼容 CJS
- Rollup: 原生 ESM 设计，Tree Shaking 更彻底
- Vite: 开发用 ESM，生产用 Rollup，开箱即用

**副作用陷阱**:
```javascript
// package.json
{ "sideEffects": ["*.css", "./src/polyfill.js"] }
```

## 5. RAG (Retrieval-Augmented Generation) 系统的完整架构是什么？如何评估检索和生成质量？

**考点**: RAG / 向量检索 / LLM 应用

**答案要点**:

**RAG 架构**:
```
离线: 文档 → 分块 → Embedding → 向量数据库

在线: 查询 → 检索相关文档 → 上下文组装 → LLM生成
```

**评估指标**:
- 检索: Recall@K, MRR, NDCG
- 生成: ROUGE, BLEU, LLM-as-Judge
- 联合: RAGAS (faithfulness, answer relevancy)

**优化方向**:
- Hybrid Retrieval（向量 + 关键词）
- Re-ranking（Cross-Encoder）
- Query 改写（HyDE）

## 8. React 的 Concurrent Mode 是如何工作的？它如何解决长任务阻塞主线程的问题？

**考点**: React 并发模式 / Scheduler / 性能优化

**答案要点**:

**问题背景**:
```
传统 React: 渲染一旦开始就必须完成，无法中断
长任务阻塞 → 用户输入响应延迟 → 卡顿体验
```

**Concurrent Mode 核心机制**:
- 优先级调度（Lane 模型）
- 时间切片（Time Slicing）
- useTransition / useDeferredValue

**代码示例**:
```javascript
function SearchResults() {
  const [query, setQuery] = useState('');
  const [isPending, startTransition] = useTransition();
  
  const handleChange = (e) => {
    const value = e.target.value;
    setQuery(value);  // 高优先级
    startTransition(() => {
      setResults(search(value));  // 低优先级，可中断
    });
  };
  // ...
}
```

## 9. CSS Container Queries 是什么？它与 Media Queries 有什么区别？

**考点**: CSS 新特性 / 响应式设计 / 容器化布局

**答案要点**:

**核心区别**:
```css
/* Media Queries - 基于视口 */
@media (max-width: 768px) { .card { flex-direction: column; } }

/* Container Queries - 基于容器 */
@container (max-width: 400px) { .card { flex-direction: column; } }
```

**使用**:
```css
.card-container { container-type: inline-size; }

@container (min-width: 400px) {
  .card { display: grid; grid-template-columns: 200px 1fr; }
}
```
