## 1. MEV-Boost 是如何工作的？PBS（Proposer-Builder Separation）对以太坊去中心化有什么影响？

**考点**: MEV / PBS / 以太坊质押

**答案要点**:

**MEV-Boost 架构**:
```
Builder (构建区块) → Relay (验证中继) → Validator (提议区块)
```

**PBS 优势**:
- 降低验证者门槛（无需 MEV 基础设施）
- 小验证者也能获得 MEV 收益

**争议**:
- Builder 中心化（Flashbots、beaverbuild 占 60%+）
- 审查风险（OFAC 合规 Builder）
- 中继单点故障

## 2. 联邦学习（Federated Learning）和同态加密（Homomorphic Encryption）在 AI 隐私保护中有什么区别？

**考点**: 隐私计算 / 联邦学习 / 同态加密

**答案要点**:

**联邦学习**:
- 数据不出本地，只交换模型更新
- 优点：通信高效、实用性强
- 缺点：梯度可能泄露训练数据

**同态加密**:
- 数据始终加密，服务器完全无法访问明文
- 优点：数学安全保证
- 缺点：计算慢 1000-10000 倍，密文膨胀

**对比**:
| 特性 | 联邦学习 | 同态加密 |
|------|----------|----------|
| 数据位置 | 本地 | 加密上传 |
| 计算成本 | 低 | 极高 |
| 隐私级别 | 中 | 高 |
| 实用性 | 高（已大规模应用） | 低（研究阶段）|

## 4. 什么是 Tree Shaking？Webpack、Rollup、Vite 在 Tree Shaking 实现上有什么区别？

**考点**: 构建工具 / 模块系统 / 代码优化

**答案要点**:

**Tree Shaking 原理**:
- 静态分析 ESM 的导出/导入
- 标记未使用代码（DCE - Dead Code Elimination）
- 依赖 ES Module 的静态结构

**工具对比**:
- Webpack: 需配置 optimization.usedExports，兼容 CJS
- Rollup: 原生 ESM 设计，Tree Shaking 更彻底
- Vite: 开发用 ESM，生产用 Rollup，开箱即用

**副作用陷阱**:
```javascript
// package.json
{ "sideEffects": ["*.css", "./src/polyfill.js"] }
```

## 6. React 的 Concurrent Mode 是如何工作的？它如何解决长任务阻塞主线程的问题？

**考点**: React 并发模式 / Scheduler / 性能优化

**答案要点**:

**问题背景**:
```
传统 React: 渲染一旦开始就必须完成，无法中断
长任务阻塞 → 用户输入响应延迟 → 卡顿体验
```

**Concurrent Mode 核心机制**:
- 优先级调度（Lane 模型）
- 时间切片（Time Slicing）
- useTransition / useDeferredValue

**代码示例**:
```javascript
function SearchResults() {
  const [query, setQuery] = useState('');
  const [isPending, startTransition] = useTransition();
  
  const handleChange = (e) => {
    const value = e.target.value;
    setQuery(value);  // 高优先级
    startTransition(() => {
      setResults(search(value));  // 低优先级，可中断
    });
  };
  // ...
}
```

## 8. Transformer 架构中的 Self-Attention 机制是如何工作的？为什么它比 RNN 更适合处理长文本？

**考点**: Transformer / Attention / 大模型架构

**答案要点**:

**Self-Attention 公式**:
```
Attention(Q, K, V) = softmax(QK^T / √d_k) · V
```

**计算过程**:
1. 将输入转换为 Q、K、V 矩阵
2. 计算注意力分数：Q × K^T
3. 缩放、Softmax 归一化
4. 加权求和：× V

**为什么比 RNN 强**:
| 特性 | RNN | Transformer |
|------|-----|-------------|
| 依赖距离 | 长距离衰减 | O(1) 直接连接 |
| 并行度 | 串行 | 全并行 |
| 训练速度 | 慢 | 快（GPU 友好）|

## 9. 什么是并行 EVM（Parallel EVM）？Monad、Sei 如何实现并行执行？

**考点**: 并行 EVM / 高性能公链 / 状态冲突

**答案要点**:

**并行执行核心问题**:
```solidity
// 交易1: Alice 给 Bob 转 100 ETH
// 交易2: Alice 给 Carol 转 50 ETH
// → 冲突！必须顺序执行

// 交易3: Bob 给 Dave 转 20 ETH
// 交易4: Carol 给 Eve 转 30 ETH  
// → 无冲突！可以并行
```

**Monad 方案**: 乐观并行执行 + 冲突检测重试
**Sei 方案**: DAG 依赖图 + 拓扑排序分组
**Fuel 方案**: UTXO 模型 + 显式输入输出声明
