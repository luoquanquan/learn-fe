## 1. React 的 Concurrent Mode 是如何工作的？它如何解决长任务阻塞主线程的问题？

**考点**: React 并发模式 / Scheduler / 性能优化

**答案要点**:

**问题背景**:
```
传统 React: 渲染一旦开始就必须完成，无法中断
长任务阻塞 → 用户输入响应延迟 → 卡顿体验
```

**Concurrent Mode 核心机制**:

```javascript
// 优先级调度 - 使用 Lane 模型
const InputContinuousHydrationLane = 0b0000000000000000000000000010000;
const DefaultLane = 0b0000000000000000000000000100000;
const TransitionLane = 0b0000000000000000000000001000000;

// 可中断的渲染
function workLoopConcurrent() {
  while (workInProgress !== null && !shouldYield()) {
    performUnitOfWork(workInProgress);
  }
  
  if (workInProgress !== null) {
    return workInProgress;
  }
}
```

**useTransition 使用**:
```javascript
function SearchResults() {
  const [query, setQuery] = useState('');
  const [isPending, startTransition] = useTransition();
  
  const handleChange = (e) => {
    const value = e.target.value;
    setQuery(value);
    startTransition(() => {
      const newResults = search(value);
      setResults(newResults);
    });
  };

    <>
      <input value={query} onChange={handleChange} />

    </>

}
```

## 3. React Server Components (RSC) 的工作原理是什么？

**考点**: React Server Components / Next.js App Router / SSR

**答案要点**:

**RSC 核心概念**:
```
传统 SSR:           RSC:
Server              Server
  ↓                   ↓
渲染 HTML          渲染 Server Component
  ↓                   ↓
hydration          流式传输 (streaming)
  ↓                   ↓
Client             Client
```

**关键区别**:
- Server Component 不打包到客户端 bundle
- 可以直接访问服务端资源（数据库、API key）
- 支持流式传输，渐进式加载

## 5. WebAssembly (WASM) 在 Web3 中有哪些应用场景？

**考点**: WebAssembly / 性能优化 / Web3 应用

**答案要点**:

**应用场景**:

1. **零知识证明计算** - 浏览器生成 zk-SNARK 证明，比 JS 快 10-100x
2. **密码学操作** - BLS 签名、椭圆曲线运算、Keccak-256 哈希
3. **链上数据解析** - RLP 解码、ABI 编码解码、交易序列化
4. **复杂计算** - AMM 价格计算、借贷协议利率模型

**Rust → WASM 示例**:
```rust
use sha3::{Keccak256, Digest};

pub fn keccak256(input: &[u8]) -> Vec<u8> {
    let mut hasher = Keccak256::new();
    hasher.update(input);
    hasher.finalize().to_vec()
}
```

## 6. 现代前端框架的响应式原理有什么区别？为什么 Signal 更优？

**考点**: 响应式系统 / 框架原理 / 性能优化

**答案要点**:

**React useState**:
- 基于 Virtual DOM + 重新渲染
- 组件级粒度，整个组件重新执行

**Vue 3 Reactivity**:
- 基于 Proxy 的依赖收集
- 属性级粒度，精确更新

**Signal 细粒度响应式**:
```javascript
// SolidJS / Preact Signals
const count = signal(0);

// 只更新 DOM 中用到的地方，不重新执行组件
effect(() => {
  console.log(count.value);
});
```

**对比**:

| 特性 | React | Vue | Signals |
|------|-------|-----|---------|
| 粒度 | 组件级 | 属性级 | 值级 |
| 性能 | 中 | 高 | 极高 |

## 8. CSS Container Queries 是什么？与 Media Queries 的区别？

**考点**: CSS 新特性 / 响应式设计 / 容器化布局

**答案要点**:

**核心区别**:
```css
/* Media Queries - 基于视口 */
@media (max-width: 768px) {
  .card { flex-direction: column; }
}

/* Container Queries - 基于容器大小 */
@container (max-width: 400px) {
  .card { flex-direction: column; }
}
```

**使用**:
```css
.card-container {
  container-type: inline-size;
}

@container (min-width: 400px) {
  .card { display: grid; grid-template-columns: 200px 1fr; }
}
```

## 9. 前端如何实现错误边界（Error Boundary）？Web3 异常如何处理？

**考点**: 错误处理 / Error Boundary / Web3 UX

**答案要点**:

**React Error Boundary**:
```javascript
class ErrorBoundary extends Component {
  static getDerivedStateFromError(error) {
    return { hasError: true };
  }
  
  componentDidCatch(error, info) {
    logErrorToService(error, info);
  }
  
  render() {
    if (this.state.hasError) {
      return <FallbackUI />;
    }
    return this.props.children;
  }
}
```

**Web3 错误分类**:
- User Rejected (4001) - 用户取消
- Insufficient Funds - Gas 不足
- Transaction Replaced - 交易被加速/取消
- Network Error - 网络切换/断开
- Contract Revert - 合约执行失败

**考点**: 微前端 / 架构设计 / Web3 状态共享
