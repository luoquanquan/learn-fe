## 1. 什么是 MPT（Merkle Patricia Trie）？以太坊的状态树、存储树、交易树有什么区别？

**考点**: 以太坊数据结构 / MPT / 状态管理

**答案要点**:

**三棵树**:
- **状态树（State Trie）**: 地址 → 账户状态（balance, nonce, codeHash, storageRoot）
- **存储树（Storage Trie）**: 每个合约独立的键值对存储
- **交易树（Tx Trie）**: 区块内所有交易
- **收据树（Receipt Trie）**: 交易执行结果和日志

**布隆过滤器**: 快速判断日志是否存在于区块中

## 2. Uniswap V3 的集中流动性（Concentrated Liquidity）是如何工作的？

**考点**: DeFi / AMM / Uniswap

**答案要点**:

**V2 vs V3**:
- V2: 流动性均匀分布在 (0, ∞)
- V3: 流动性可集中在特定价格区间

**数学原理**:
- 虚拟流动性：仅在设定区间内有效
- 资金效率提升：同样资金可支持更大交易量
- 无常损失：区间越窄，风险越高

**Position NFT**: 流动性位置可转让、可组合

## 7. 什么是 WebAssembly？在浏览器中如何实现与 JavaScript 的高性能交互？

**考点**: WebAssembly / WASM / 性能优化

**答案要点**:

**WASM 优势**:
- 接近原生性能（比 JS 快 10-100x）
- 语言无关（C++/Rust/Go 都可编译）
- 沙箱安全

**与 JS 交互**:
```javascript
// 导出 WASM 函数
const wasm = await WebAssembly.instantiate(module, imports);
const result = wasm.exports.add(1, 2); // 直接调用

// 共享内存
const memory = new WebAssembly.Memory({ initial: 10 });
const bytes = new Uint8Array(memory.buffer);
```

**Web3 应用**: zk-SNARK 证明生成、BLS 签名聚合

## 10. 浏览器渲染流程是什么？什么是关键渲染路径（CRP）优化？

**考点**: 浏览器原理 / 渲染流程 / 性能优化

**答案要点**:

**渲染流程**:
1. 解析 HTML → DOM 树
2. 解析 CSS → CSSOM 树  
3. DOM + CSSOM → Render 树
4. Layout（计算几何信息）
5. Paint（绘制像素）
6. Composite（合成图层）

**CRP 优化**:
- CSS 放头部，避免阻塞渲染
- JS 放底部或 defer/async
- 减少重排（Layout），使用 transform/opacity 动画
- 字体预加载：rel="preload"

**Web3 DApp**: 钱包连接弹窗优先加载，背景数据延后
