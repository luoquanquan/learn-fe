## 1. 硬件钱包（Ledger/Trezor）与软件钱包的集成方案？如何实现安全通信？

**考点**: 硬件钱包 / Ledger / 安全通信 / 离线签名

**答案要点**:

**通信方式**:
- USB HID / WebUSB
- Bluetooth（Ledger Nano X）
- QRCode（空气间隙）

**集成流程**:
```
1. 连接硬件钱包
2. 获取公钥/地址（派生路径 m/44'/60'/0'/0/0）
3. 构建交易并发送到硬件设备
4. 用户在设备上确认
5. 设备返回签名
6. 前端广播交易
```

**安全要点**:
- 所有敏感操作在硬件内完成
- 显示地址在硬件屏幕上确认
- 固件版本检查

## 3. 钱包的新手引导（Onboarding）流程如何设计？如何平衡安全教育和用户体验？

**考点**: 新手引导 / 用户体验 / 安全教育 / Onboarding

**答案要点**:

**引导流程**:
```
1. 助记词创建/导入
   ↓ 强调备份重要性
2. 安全提示（截图警告、泄露风险）
   ↓
3. 简单交互教程（收款、发送）
   ↓
4. 首次交易引导
```

**平衡策略**:
- 分步教育，不一次性灌输
- 危险操作强制确认（如展示私钥）
- 可跳过，但风险明确提示
- 后续随时可查看安全中心

## 5. 钱包的 KYT（Know Your Transaction）和合规功能如何实现？

**考点**: 合规 / KYT / 反洗钱 / 监管

**答案要点**:

**KYT 功能**:
- 交易对手风险评分
- 混币器/暗网地址标记

**数据源**:
- Chainalysis
- Elliptic
- TRM Labs
- 平台自有风控系统

**用户提示**:
- 风险地址警告
- 资金来源说明要求

## 7. 钱包的 API 密钥和第三方服务如何安全管理？防止密钥泄露的最佳实践？

**考点**: API密钥 / 安全管理 / 密钥泄露 / 第三方服务

**答案要点**:

**密钥管理**:
- 不在前端代码硬编码
- 使用环境变量 + CI/CD 注入
- 密钥轮换机制

**防护措施**:
```
1. API 密钥权限最小化
2. IP 白名单限制
3. 请求频率限制
4. 异常行为监控
```

**泄露响应**:
- 立即撤销密钥
- 审计日志检查
- 通知受影响用户

## 9. 如何实现钱包的链上消息签名功能？EIP-712 结构化数据签名有什么优势？

**考点**: 消息签名 / EIP-712 / 结构化数据 / 安全

**答案要点**:

**签名类型**:
- personal_sign：简单消息，可读性差
- signTypedData_v4（EIP-712）：结构化，可读

**EIP-712 优势**:
```javascript
const domain = {
  name: 'My Dapp',
  version: '1',
  chainId: 1,
  verifyingContract: '0x...'
};

// 用户看到的:
// 签名请求来自 My Dapp
// 链: Ethereum
// 内容: { amount: 100, token: USDT }
```

**防范**：防止签名钓鱼（如 OpenSea 钓鱼事件）
