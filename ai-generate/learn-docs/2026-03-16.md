## 1. JavaScript 异步编程与 Promise 原理【前端】

**考点**: JavaScript / Promise / async/await / 事件循环

**答案要点**:

### Promise 状态

| 状态 | 含义 | 转换 |
|------|------|------|
| **pending** | 等待中 | 初始状态 |
| **fulfilled** | 已完成 | `resolve()` |
| **rejected** | 已拒绝 | `reject()` |

### Promise 链式调用

```javascript
fetch('/api/user')
  .then(response => response.json())
  .then(data => {
    console.log(data);
    return fetch(`/api/orders/${data.id}`);
  })
  .then(response => response.json())
  .catch(error => console.error(error))
  .finally(() => console.log('完成'));
```

### async/await

```javascript
async function getUserData() {
  try {
    const user = await fetch('/api/user').then(r => r.json());
    const orders = await fetch(`/api/orders/${user.id}`).then(r => r.json());
    return { user, orders };
  } catch (error) {
    console.error('获取失败:', error);
    throw error;
  }
}
```

### Promise.all vs Promise.race

```javascript
// 全部完成
const [users, posts] = await Promise.all([
  fetch('/api/users'),
  fetch('/api/posts')
]);

// 竞速，取最快
const fastest = await Promise.race([
  fetch('/api/a'),
  fetch('/api/b')
]);
```

## 4. 去中心化身份（DID）与 ENS【Web3】

**考点**: DID / ENS / 身份验证 / 去中心化域名

**答案要点**:

### ENS 以太坊域名服务

```
0x742d35Cc6634C0532925a3b844Bc9e7595f0bEb
    ↓
vitalik.eth
```

### ENS 解析

```javascript
const ethers = require('ethers');
const provider = ethers.getDefaultProvider();

// 正向解析：ENS -> 地址
const address = await provider.resolveName('vitalik.eth');
console.log(address); // 0x742d35...

// 反向解析：地址 -> ENS
const ensName = await provider.lookupAddress('0x742d35...');
console.log(ensName); // vitalik.eth
```

### DID 架构

```
DID Document
├── id: did:ethr:0x123...
├── verificationMethod: 公钥
├── authentication: 认证方式
└── service: 服务端点
```

### 应用场景

- **简化转账**：输入 `alice.eth` 而非长地址
- **身份展示**：Web3 社交-profile
- **多链统一**：一个 ENS 绑定多链地址

## 5. MEV（最大可提取价值）与三明治攻击【Web3】

**考点**: MEV / 三明治攻击 / 抢跑 / 矿工可提取价值

**答案要点**:

### 什么是 MEV

**MEV（Maximal Extractable Value）**：矿工/验证者通过重新排序、插入或审查交易可提取的最大价值。

### MEV 类型

| 类型 | 说明 | 例子 |
|------|------|------|
| **抢跑（Frontrunning）** | 看到大买单，抢先买入 | DEX 交易 |
| **尾随（Backrunning）** | 大交易后套利 | 价格修正 |
| **三明治攻击** | 前后夹击目标交易 | 夹心攻击 |
| **清算** | 清算抵押不足的仓位 | 借贷协议 |

### 三明治攻击流程

```
攻击者买入 → 推高价格
    ↓
受害者大额买入 → 价格进一步上涨
    ↓
攻击者卖出 → 获利
```

### 防护措施

1. **使用私有内存池**：Flashbots Protect
2. **滑点保护**：设置最大滑点
3. **分批交易**：避免大额单笔交易
4. **MEV 保险**：CowSwap、MEV Blocker

```javascript
// Flashbots Protect RPC
const provider = new ethers.JsonRpcProvider(
  'https://rpc.flashbots.net'
);
```

## 7. 生成对抗网络（GAN）原理与应用【AI】

**考点**: 深度学习 / GAN / 生成模型 / 对抗训练

**答案要点**:

### GAN 架构

```
随机噪声 z → Generator → 假数据
                           ↓
真数据 → Discriminator → 真假判断
```

### 训练过程

| 阶段 | 优化器 | 目标 |
|------|--------|------|
| **训练 D** | 最大化 | 正确区分真假 |
| **训练 G** | 最小化 | 欺骗判别器 |

### 损失函数

```
min_G max_D V(D, G) = E[log D(x)] + E[log(1 - D(G(z)))]

D(x): 判别器判断真数据的概率
G(z): 生成器从噪声生成的数据
```

### 经典 GAN 变体

| 模型 | 特点 | 应用 |
|------|------|------|
| **DCGAN** | 深度卷积 GAN | 图像生成 |
| **StyleGAN** | 控制生成风格 | 人脸生成 |
| **CycleGAN** | 无配对数据 | 风格迁移 |
| **Pix2Pix** | 条件生成 | 图像转换 |

### 应用场景

- **图像生成**：This Person Does Not Exist
- **风格迁移**：照片转油画
- **数据增强**：生成训练数据
- **超分辨率**：图像放大

## 9. 响应式设计与移动端适配【前端】

**考点**: 响应式 / 移动端 / Viewport / 媒体查询 / Rem

**答案要点**:

### Viewport 设置

```html
<meta name="viewport" content="width=device-width, initial-scale=1.0">
```

### 媒体查询

```css
/* 移动端优先 */
.container {
  width: 100%;
}

/* 平板 */
@media (min-width: 768px) {
  .container {
    width: 750px;
  }
}

/* 桌面 */
@media (min-width: 1024px) {
  .container {
    width: 960px;
  }
}
```

### 适配方案对比

| 方案 | 原理 | 优点 | 缺点 |
|------|------|------|------|
| **Rem** | 根字体大小缩放 | 兼容性好 | 需要计算 |
| **VW/VH** | 视口单位 | 直接使用 | 兼容性略差 |
| **Flex** | 弹性布局 | 简单 | 复杂布局受限 |
| **Scale** | 整体缩放 | 还原设计稿 | 字体模糊 |

### Rem 适配

```javascript
// 设置根字体大小
function setRem() {
  const baseWidth = 375; // 设计稿宽度
  const currentWidth = document.documentElement.clientWidth;
  const scale = currentWidth / baseWidth;
  document.documentElement.style.fontSize = `${scale * 100}px`;
}

// 使用
.box {
  width: 1.75rem; /* 175px */
}
```

## 10. 微前端架构与实践【前端】

**考点**: 微前端 / qiankun / Module Federation / 架构设计

**答案要点**:

### 微前端核心价值

| 价值 | 说明 |
|------|------|
| **技术栈无关** | 各团队自主选择技术栈 |
| **独立开发** | 独立部署、独立运行 |
| **渐进升级** | 逐步改造遗留系统 |
| **团队自治** | 减少团队间依赖 |

### 实现方案对比

| 方案 | 原理 | 代表 |
|------|------|------|
| **iframe** | 原生隔离 | 传统方案 |
| **qiankun** | JS Sandbox + HTML Entry | 阿里开源 |
| **Module Federation** | 运行时模块共享 | Webpack 5 |
| **Micro App** | 类 WebComponent | 京东开源 |

### qiankun 使用

```javascript
import { registerMicroApps, start } from 'qiankun';

registerMicroApps([
  {
    name: 'reactApp',
    entry: '//localhost:3000',
    container: '#container',
    activeRule: '/react',
  },
  {
    name: 'vueApp',
    entry: '//localhost:8080',
    container: '#container',
    activeRule: '/vue',
  },
]);

start();
```

### 通信机制

```javascript
// 主应用
import { initGlobalState } from 'qiankun';

const actions = initGlobalState({ user: null });

// 子应用
export async function mount(props) {
  props.onGlobalStateChange((state, prev) => {
    console.log('全局状态变化:', state);
  });
  
  props.setGlobalState({ user: { name: 'Alice' } });
}
```
