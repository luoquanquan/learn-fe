## 1. TypeScript 高级类型体操：实现一个类型安全的 DeepReadonly

**考点**: TypeScript、高级类型、泛型、递归类型、条件类型

**答案要点**:

### 核心原理解释

在 Web3 钱包开发中，我们经常需要处理复杂的配置对象和区块链数据，这些数据在传入后不应该被意外修改。TypeScript 提供了 `readonly` 修饰符，但它只作用于第一层属性。要实现深度只读（DeepReadonly），需要使用递归泛型。

DeepReadonly 的核心思想是递归遍历对象的所有层级，将每一层的属性都标记为 readonly。这涉及到 TypeScript 的几个高级特性：
1. **递归泛型**：类型定义中引用自身
2. **条件类型**：使用 `extends` 判断类型
3. **映射类型**：使用 `in` 关键字遍历属性
4. **内置工具类型**：如 `keyof`、`ReturnType` 等

### 代码示例

```typescript
// 基础版 DeepReadonly
type DeepReadonly<T> = {
  readonly [K in keyof T]: T[K] extends object 
    ? T[K] extends Function 
      ? T[K] 
      : DeepReadonly<T[K]>
    : T[K];
};

// 完整版（处理数组、Map、Set 等）
type DeepReadonly<T> = T extends (infer R)[] 
  ? ReadonlyArray<DeepReadonly<R>>
  : T extends Map<infer K, infer V>
  ? ReadonlyMap<DeepReadonly<K>, DeepReadonly<V>>
  : T extends Set<infer S>
  ? ReadonlySet<DeepReadonly<S>>
  : T extends object
  ? { readonly [K in keyof T]: DeepReadonly<T[K]> }
  : T;

// 使用场景：Web3 钱包配置
type WalletConfig = {
  chains: {
    id: number;
    rpc: string[];
    tokens: Array<{ symbol: string; decimals: number }>;
  }[];
  features: {
    hardwareWallet: {
      supported: string[];
      config: Record<string, unknown>;
    };
  };
};

type FrozenWalletConfig = DeepReadonly<WalletConfig>;

// 实际应用
const config: FrozenWalletConfig = {
  chains: [{ id: 1, rpc: ["https://..."], tokens: [] }],
  features: { hardwareWallet: { supported: ["Ledger"], config: {} } }
};

// 以下操作都会报错
// config.chains[0].id = 2; // Error
// config.features.hardwareWallet.supported.push("Trezor"); // Error
```

1. **配置管理**：将链配置、代币列表等标记为 DeepReadonly，防止运行时被篡改
2. **状态管理**：在 Redux/Vuex 中，确保 state 的不可变性
3. **API 响应**：将后端返回的数据标记为只读，避免前端意外修改

### 常见问题和解决方案

1. **循环引用问题**：如果类型有循环引用，会导致类型递归过深
   - 解决：使用 `interface` 配合类型断言，或限制递归深度

2. **性能问题**：深层嵌套的类型会导致编译变慢
   - 解决：对关键路径使用，而非全局应用

3. **与 JSON 序列化的冲突**：DeepReadonly 类型不能直接被 JSON.stringify 处理
   - 解决：使用类型转换 `JSON.parse(JSON.stringify(obj))`

## 2. 前端性能优化：高并发场景下的内存管理与 OOM 防护

**考点**: 性能优化、内存管理、垃圾回收、事件循环、流式处理

**答案要点**:

### 核心原理解释

在快手「回森」项目中，高并发弹幕导致的 OOM 问题是一个典型的前端性能挑战。核心问题包括：

1. **内存泄漏**：事件监听器未正确移除、闭包持有大对象
2. **垃圾回收机制**：V8 的分代回收策略，以及何时触发 Major GC
3. **事件循环阻塞**：大量同步操作导致主线程卡顿
4. **DOM 节点累积**：大量 DOM 节点占用内存

解决思路是**流式处理 + 对象池 + 虚拟滚动**的组合策略。

### 代码示例

```typescript
// 1. 对象池模式复用弹幕对象
class DanmakuPool {
  private pool: DanmakuItem[] = [];
  private maxSize = 100;
  
  acquire(data: DanmakuData): DanmakuItem {
    const item = this.pool.pop() || new DanmakuItem();
    item.reset(data);
    return item;
  }
  
  release(item: DanmakuItem): void {
    if (this.pool.length < this.maxSize) {
      item.clear();
      this.pool.push(item);
    }
  }
}

// 2. 流式处理 + 反压控制
class DanmakuStream {
  private buffer: DanmakuData[] = [];
  private processing = false;
  private maxBuffer = 500;
  
  push(data: DanmakuData): void {
    if (this.buffer.length >= this.maxBuffer) {
      // 反压：丢弃最早的 20%
      this.buffer = this.buffer.slice(this.maxBuffer * 0.2);
    }
    this.buffer.push(data);
    this.process();
  }
  
  private async process(): Promise<void> {
    if (this.processing) return;
    this.processing = true;
    
    while (this.buffer.length > 0) {
      const batch = this.buffer.splice(0, 50); // 每批处理50条
      await this.renderBatch(batch);
      
      // 让出主线程
      await new Promise(resolve => requestAnimationFrame(resolve));
    }
    
    this.processing = false;
  }
}

// 3. 虚拟滚动 + Canvas 渲染
class DanmakuRenderer {
  private canvas: HTMLCanvasElement;
  private ctx: CanvasRenderingContext2D;
  private visibleTracks: Map<number, DanmakuItem[]> = new Map();
  
  render(items: DanmakuItem[]): void {
    // 只渲染可视区域内的弹幕
    const visibleItems = items.filter(item => 
      item.y > this.scrollTop - 100 && 
      item.y < this.scrollTop + this.viewportHeight + 100
    );
    
    this.ctx.clearRect(0, 0, this.canvas.width, this.canvas.height);
    
    for (const item of visibleItems) {
      this.drawDanmaku(item);
    }
  }
  
  private drawDanmaku(item: DanmakuItem): void {
    this.ctx.fillStyle = item.color;
    this.ctx.font = `${item.fontSize}px Arial`;
    this.ctx.fillText(item.text, item.x, item.y);
  }
}
```

## 3. 硬件钱包接入：Ledger/Trezor 的通信协议与安全模型

**考点**: 硬件钱包、USB HID、协议设计、端到端加密、安全模型

**答案要点**:

### 核心原理解释

硬件钱包（Cold Wallet）的核心安全模型是**私钥永不触网**。以 Ledger 为例：

1. **物理隔离**：私钥存储在 Secure Element（安全芯片）中
2. **可信显示**：交易详情在设备屏幕上显示，防止中间人攻击
3. **用户确认**：物理按钮确认签名，防止恶意软件自动签名
4. **分层确定性钱包**：遵循 BIP-32/39/44 标准，一个种子派生多个地址

通信协议通常使用 **APDU（Application Protocol Data Unit）** 格式，通过 USB HID 或 WebUSB 与主机通信。

### 代码示例

```typescript
// Ledger 硬件钱包接入封装
import TransportWebUSB from "@ledgerhq/hw-transport-webusb";
import AppEth from "@ledgerhq/hw-app-eth";

class LedgerConnector {
  private transport: TransportWebUSB | null = null;
  private app: AppEth | null = null;
  
  async connect(): Promise<string> {
    // 请求 USB 权限
    this.transport = await TransportWebUSB.create();
    this.app = new AppEth(this.transport);
    
    // 获取地址（不暴露私钥）
    const { address, publicKey } = await this.app.getAddress(
      "44'/60'/0'/0/0",  // BIP-44 路径
      false,             // 不在设备上显示
      true               // 返回公钥
    );
    
    return address;
  }
  
  async signTransaction(txData: TransactionRequest): Promise<string> {
    if (!this.app) throw new Error("未连接");
    
    // 序列化交易
    const tx = Transaction.from(txData);
    const serialized = tx.unsignedSerialized;
    
    // 发送给 Ledger 签名（用户需在设备上确认）
    const signature = await this.app.signTransaction(
      "44'/60'/0'/0/0",
      serialized.slice(2) // 移除 0x 前缀
    );
    
    // 组合签名和交易
    return this.combineSignature(tx, signature);
  }
  
  async signMessage(message: string): Promise<string> {
    if (!this.app) throw new Error("未连接");
    
    // 计算消息哈希
    const messageHex = Buffer.from(message).toString('hex');
    
    // 设备显示消息内容并请求签名
    const signature = await this.app.signPersonalMessage(
      "44'/60'/0'/0/0",
      messageHex
    );
    
    return `0x${signature.r}${signature.s}${signature.v.toString(16)}`;
  }
}

// 统一的硬件钱包管理器
export class HardwareWalletManager {
  private connectors: Map<string, HWConnector> = new Map();
  
  constructor() {
    this.connectors.set('ledger', new LedgerConnector());
    this.connectors.set('trezor', new TrezorConnector());
    this.connectors.set('onekey', new OneKeyConnector());
    this.connectors.set('keystone', new KeystoneConnector());
  }
  
  async connect(type: string): Promise<HWAccount> {
    const connector = this.connectors.get(type);
    if (!connector) throw new Error(`不支持的硬件钱包: ${type}`);
    
    const address = await connector.connect();
    
    return {
      address,
      type,
      connector,
      signTransaction: (tx) => connector.signTransaction(tx),
      signMessage: (msg) => connector.signMessage(msg),
      signTypedData: (data) => connector.signTypedData(data) // EIP-712
    };
  }
}
```

## 4. 多链交易解析：EVM/Solana/Tron 交易结构差异与解析策略

**考点**: 多链架构、交易解析、ABI 解码、数据序列化、安全防护

**答案要点**:

### 核心原理解释

多链钱包需要解析不同区块链的交易，主要差异：

| 特性 | EVM | Solana | Tron |
|------|-----|--------|------|
| 交易结构 | RLP 编码 | 二进制 (bincode) | protobuf |
| 指令模型 | 智能合约调用 | 多指令数组 | 单一合约调用 |
| 费用模型 | Gas Price/Limit | 计算单元/租金 | 带宽/能量 |
| 地址格式 | 0x... | Base58 | Base58Check |

**解析策略**：
1. **统一抽象层**：定义链无关的交易数据结构
2. **链特定解析器**：针对每条链实现解析逻辑
3. **ABI/IDL 管理**：维护合约接口定义
4. **安全检测**：识别风险操作（授权、转账等）

### 代码示例

```typescript
// 统一的交易数据结构
interface ParsedTransaction {
  chainType: ChainType;
  from: string;
  to: string;
  value: BigNumber;
  data?: HexString;
  parsedData?: {
    method?: string;
    params: Record<string, unknown>;
    riskLevel: 'safe' | 'warning' | 'dangerous';
  };
  gasInfo: {
    limit: BigNumber;
    price: BigNumber;
    estimatedCost: BigNumber;
  };
}

// EVM 交易解析器
class EVMTransactionParser {
  private abiDecoder = new AbiDecoder();
  
  parse(rawTx: string): ParsedTransaction {
    // RLP 解码
    const decoded = RLP.decode(rawTx);
    const [nonce, gasPrice, gasLimit, to, value, data, v, r, s] = decoded;
    
    // EIP-1559 处理
    const isEIP1559 = decoded.length === 9 && decoded[0] === '0x02';
    
    // 解析合约调用
    let parsedData;
    if (data && data !== '0x') {
      parsedData = this.parseContractData(data, to);
    }
    
    return {
      chainType: 'evm',
      from: this.recoverFrom(v, r, s),
      to,
      value: BigNumber.from(value),
      data,
      parsedData,
      gasInfo: {
        limit: BigNumber.from(gasLimit),
        price: BigNumber.from(gasPrice),
        estimatedCost: this.calculateGasCost(gasPrice, gasLimit)
      }
    };
  }
  
  private parseContractData(data: string, to: string): ParsedData {
    const methodId = data.slice(0, 10);
    const abi = this.getABI(to);
    
    // 识别常见风险操作
    const riskSignatures: Record<string, string> = {
      '0x095ea7b3': 'approve',      // 授权代币
      '0xa9059cbb': 'transfer',     // 转账
      '0x23b872dd': 'transferFrom', // 代转账
      '0x8da5cb5b': 'owner',        // 查询所有者（可能涉及权限）
    };
    
    const methodName = riskSignatures[methodId] || 'unknown';
    
    // 解码参数
    const params = abi ? this.abiDecoder.decode(abi, data) : {};
    
    // 风险评估
    const riskLevel = this.assessRisk(methodName, params);
    
    return {
      method: methodName,
      params,
      riskLevel
    };
  }
  
  private assessRisk(method: string, params: any): RiskLevel {
    // approve 操作检查授权额度
    if (method === 'approve') {
      const amount = params.amount || params.value;
      if (amount === '0xffffffffffffffffffffffffffffffffffffffff') {
        return 'dangerous'; // 无限授权
      }
    }
    
    // transferFrom 检查是否有授权
    if (method === 'transferFrom') {
      // 需要检查 allowance
    }
    
    return 'warning';
  }
}

// Solana 交易解析器
class SolanaTransactionParser {
  parse(buffer: Buffer): ParsedTransaction {
    // Solana 使用 bincode 序列化
    const transaction = Transaction.from(buffer);
    
    // 解析指令
    const instructions = transaction.instructions.map(ix => {
      const program = this.identifyProgram(ix.programId);
      
      return {
        program,
        accounts: ix.keys.map(k => k.pubkey.toBase58()),
        data: this.decodeInstructionData(program, ix.data)
      };
    });
    
    // 风险评估
    const riskLevel = this.assessSolanaRisk(instructions);
    
    return {
      chainType: 'solana',
      from: transaction.feePayer?.toBase58() || '',
      to: instructions[0]?.accounts[0] || '',
      value: BigNumber.from(0),
      parsedData: {
        method: instructions.map(i => i.program).join(','),
        params: { instructions },
        riskLevel
      },
      gasInfo: {
        limit: BigNumber.from(0),
        price: BigNumber.from(5000), // Lamports per signature
        estimatedCost: BigNumber.from(instructions.length * 5000)
      }
    };
  }
  
  private identifyProgram(programId: PublicKey): string {
    const knownPrograms: Record<string, string> = {
      '11111111111111111111111111111111': 'system',
      'TokenkegQfeZyiNwAJbNbGKPFXCWuBvf9Ss623VQ5DA': 'token',
      'ATokenGPvbdGVxr1b2hvZbsiqW5xWH25efTNsLJA8knL': 'associated_token',
    };
    return knownPrograms[programId.toBase58()] || 'unknown';
  }
}

export class TransactionParserFactory {

    this.parsers.set('tron', new TronTransactionParser());

  parse(chain: ChainType, rawTx: string | Buffer): ParsedTransaction {
    const parser = this.parsers.get(chain);
    if (!parser) throw new Error(`不支持的链: ${chain}`);
    return parser.parse(rawTx);
  }
}
```

## 5. EIP-7702 与 Permit2：新型签名场景的安全风险分析

**考点**: EIP 标准、无 Gas 交易、签名安全、权限管理、风险防护

**答案要点**:

### 核心原理解释

**EIP-7702（Set Code for EOAs）**：允许 EOA 账户设置代码地址，实现账户抽象（Account Abstraction）的过渡方案。

**Permit2**：Uniswap 推出的代币授权管理合约，解决 ERC20 的 `approve` 模式问题：
- 无限授权风险
- 每笔交易都需要先授权
- 无法离线签名授权

安全风险主要来自：
1. **签名钓鱼**：诱导用户签署恶意 Permit/EIP-712 消息
2. **权限升级**：恶意合约获取账户控制权
3. **重放攻击**：签名在其他链或场景被重用

```typescript
class SignatureSecurityChecker {
  // 检测签名是否涉及敏感操作
  async checkSignatureRisk(
    message: string | TypedData,
    signature: string
  ): Promise<RiskAssessment> {
    // 1. 解析签名内容
    const parsed = this.parseSignature(message);
    
    // 2. 检查是否为 Permit 类型
    if (this.isPermitSignature(parsed)) {
      return this.assessPermitRisk(parsed);
    }
    
    // 3. 检查是否为 EIP-7702 设置代码
    if (this.isEIP7702Signature(parsed)) {
      return this.assessEIP7702Risk(parsed);
    }
    
    // 4. 检查是否涉及资产转移
    if (this.isTransferSignature(parsed)) {
      return { level: 'high', reason: '可能涉及资产转移' };
    }
    
    return { level: 'low', reason: '常规消息签名' };
  }
  
  private assessPermitRisk(permitData: any): RiskAssessment {
    const { token, amount, spender } = permitData;
    
    // 检查 spender 是否为已知合约
    const spenderReputation = this.checkContractReputation(spender);
    if (spenderReputation.risky) {
      return {
        level: 'dangerous',
        reason: `授权给风险合约: ${spenderReputation.reason}`
      };
    }
    
    // 检查授权金额
    if (amount === MaxUint160) {
      return {
        level: 'high',
        reason: '无限授权风险，可能被利用转移所有资产'
      };
    }
    
    // 检查代币
    const tokenInfo = this.getTokenInfo(token);
    if (tokenInfo.isHoneypot) {
      return {
        level: 'dangerous',
        reason: '该代币被标记为蜜罐合约'
      };
    }
    
    return { level: 'medium', reason: '代币授权操作' };
  }
  
  private assessEIP7702Risk(eip7702Data: any): RiskAssessment {
    const { codeAddress } = eip7702Data;
    
    // 检查代码地址
    const contractAnalysis = this.analyzeContract(codeAddress);
    
    if (contractAnalysis.canSelfDestruct) {
      return {
        level: 'dangerous',
        reason: '代码合约包含自毁功能，可能导致资产丢失'
      };
    }
    
    if (contractAnalysis.hasAdminFunction) {
      return {
        level: 'high',
        reason: '代码合约有管理员功能，可能控制您的账户'
      };
    }
    
    return {
      level: 'medium',
      reason: 'EIP-7702 代码设置，请确认您信任此合约'
    };
  }
  
  // 检查合约信誉
  private checkContractReputation(address: string): ContractReputation {
    // 查询数据库/API
    const blacklist = this.getBlacklist();
    const whitelist = this.getWhitelist();
    
    if (blacklist.includes(address)) {
      return { risky: true, reason: '地址在黑名单中' };
    }
    
    if (whitelist.includes(address)) {
      return { risky: false, reason: '知名协议' };
    }
    
    // 检查合约代码
    return this.analyzeContractCode(address);
  }
}
```

## 6. 多链钱包架构：BTC/Solana/Sui 等非 EVM 链的统一抽象设计

**考点**: 多链架构、抽象设计、账户模型、密钥派生、统一接口

**答案要点**:

### 核心原理解释

多链钱包面临的核心挑战是**不同链的账户模型差异**：

| 链 | 账户模型 | 地址生成 | 签名算法 |
|----|----------|----------|----------|
| EVM | 账户模型 | Keccak256(pubkey) | ECDSA (secp256k1) |
| BTC | UTXO | Hash160(pubkey) | ECDSA (secp256k1) |
| Solana | 账户模型 | Base58(pubkey) | Ed25519 |
| Sui | 对象模型 | Blake2b(pubkey) | Ed25519 |
| Aptos | 账户模型 | SHA3-256(pubkey) | Ed25519 |

**设计原则**：
1. **统一账户抽象**：用户视角的单一账户，底层映射到多链地址
2. **分层确定性钱包**：一个种子派生所有链的私钥
3. **链无关接口**：上层业务不感知链差异
4. **插件化扩展**：新链通过插件接入

### 代码示例

```typescript
// 统一账户抽象
interface UnifiedAccount {
  id: string;           // 用户可见的统一 ID
  name: string;         // 账户名称
  avatar?: string;      // 头像
  
  // 各链地址（派生自同一主密钥）
  addresses: {
    evm: AddressInfo;      // Ethereum, BSC, Polygon...
    bitcoin: AddressInfo;  // BTC (支持多地址类型)
    solana: AddressInfo;
    sui: AddressInfo;
    aptos: AddressInfo;
    tron: AddressInfo;
  };
  
  // 派生配置
  derivation: {
    seed: string;         // 加密存储的种子
    pathTemplate: string; // 派生路径模板
  };
}

interface AddressInfo {
  address: string;
  publicKey: string;
  derivationPath: string;
  addressType?: string;   // BTC: legacy/segwit/taproot
}

// 密钥派生管理器
class KeyDerivationManager {
  private seed: Buffer;
  
  constructor(mnemonic: string, password?: string) {
    this.seed = bip39.mnemonicToSeedSync(mnemonic, password);
  }
  
  // EVM 派生 (BIP-44)
  deriveEVMAddress(index: number = 0): AddressInfo {
    const path = `m/44'/60'/0'/0/${index}`;
    const node = HDNode.fromSeed(this.seed).derivePath(path);
    
    return {
      address: ethers.utils.computeAddress(node.publicKey),
      publicKey: node.publicKey,
      derivationPath: path
    };
  }
  
  // BTC 派生 (支持多地址类型)
  deriveBTCAddress(index: number = 0, type: 'legacy' | 'segwit' | 'taproot' = 'segwit'): AddressInfo {
    const coinType = 0; // BTC
    const path = `m/84'/${coinType}'/0'/0/${index}`; // BIP-84 for segwit
    
    const node = HDNode.fromSeed(this.seed).derivePath(path);
    
    let address: string;
    switch (type) {
      case 'legacy':
        address = bitcoin.payments.p2pkh({ 
          pubkey: Buffer.from(node.publicKey.slice(2), 'hex') 
        }).address!;
        break;
      case 'segwit':
        address = bitcoin.payments.p2wpkh({ 
          pubkey: Buffer.from(node.publicKey.slice(2), 'hex') 
        }).address!;
        break;
      case 'taproot':
        address = bitcoin.payments.p2tr({ 
          internalPubkey: Buffer.from(node.publicKey.slice(2), 'hex').slice(1)
        }).address!;
        break;
    }
    
    return {
      address,
      publicKey: node.publicKey,
      derivationPath: path,
      addressType: type
    };
  }
  
  // Solana 派生 (Ed25519)
  deriveSolanaAddress(index: number = 0): AddressInfo {
    // Solana 使用 ed25519，路径格式不同
    const path = `m/44'/501'/${index}'/0'`;
    
    const seed = derivePath(path, this.seed.toString('hex')).key;
    const keypair = nacl.sign.keyPair.fromSeed(seed);
    
    return {
      address: bs58.encode(keypair.publicKey),
      publicKey: Buffer.from(keypair.publicKey).toString('hex'),
      derivationPath: path
    };
  }
  
  // Sui 派生
  deriveSuiAddress(index: number = 0): AddressInfo {
    const path = `m/44'/784'/0'/0'/${index}'`;
    
    const seed = derivePath(path, this.seed.toString('hex')).key;
    const keypair = nacl.sign.keyPair.fromSeed(seed);
    
    // Sui 地址是 0x + publicKey 的 Blake2b hash
    const hash = blake2b(keypair.publicKey, undefined, 32);
    
    return {
      address: '0x' + Buffer.from(hash).toString('hex'),
      publicKey: Buffer.from(keypair.publicKey).toString('hex'),
      derivationPath: path
    };
  }
}

  signMessage(message: string | Uint8Array): Promise<string>;
  getAddress(): string;

class UnifiedSignerFactory {
  constructor(private keyManager: KeyDerivationManager) {}
  
  createSigner(chain: ChainType, addressIndex: number = 0): ChainSigner {
    switch (chain) {
      case 'evm':
        return new EVMSigner(this.keyManager, addressIndex);
      case 'bitcoin':
        return new BTCSigner(this.keyManager, addressIndex);
      case 'solana':
        return new SolanaSigner(this.keyManager, addressIndex);
      case 'sui':

    }
  }

// EVM 签名器实现
class EVMSigner implements ChainSigner {
  private wallet: ethers.Wallet;
  
  constructor(keyManager: KeyDerivationManager, index: number) {
    const { privateKey } = keyManager.deriveEVMAddress(index);
    this.wallet = new ethers.Wallet(privateKey);
  }
  
  async signTransaction(tx: ethers.TransactionRequest): Promise<string> {
    return this.wallet.signTransaction(tx);
  }
  
  async signMessage(message: string): Promise<string> {
    return this.wallet.signMessage(message);
  }
  
  getAddress(): string {
    return this.wallet.address;
  }
}

// Solana 签名器实现
class SolanaSigner implements ChainSigner {
  private keypair: Keypair;
  
  constructor(keyManager: KeyDerivationManager, index: number) {
    const { derivationPath } = keyManager.deriveSolanaAddress(index);
    const seed = derivePath(derivationPath, keyManager.seed.toString('hex')).key;
    this.keypair = Keypair.fromSeed(seed);
  }
  
  async signTransaction(tx: Transaction): Promise<string> {
    tx.sign(this.keypair);
    return tx.serialize().toString('base64');
  }
  
  async signMessage(message: Uint8Array): Promise<string> {
    const signature = nacl.sign.detached(message, this.keypair.secretKey);
    return bs58.encode(signature);
  }
  
  getAddress(): string {
    return this.keypair.publicKey.toBase58();
  }
}
```

## 7. 钱包安全防护：钓鱼识别、合约风险检测与签名安全

**考点**: Web3 安全、钓鱼检测、合约审计、签名验证、风险评分

**答案要点**:

### 核心原理解释

Web3 钱包面临的主要安全威胁：

1. **钓鱼攻击**：
   - 假网站诱导连接钱包
   - 假客服索要助记词
   - 假空投诱导授权

2. **合约风险**：
   - 蜜罐合约（Honeypot）
   - 代理合约恶意升级
   - 隐藏的后门函数

3. **签名风险**：
   - eth_sign 盲签
   - 恶意 Permit 授权
   - 结构化数据签名欺骗

**防护体系**：
- **事前检测**：域名/合约/代币风险评分
- **事中拦截**：交易签名前风险识别
- **事后保护**：授权管理 + 资产监控

### 代码示例

```typescript
// 钓鱼检测服务
class PhishingDetector {
  private domainBlacklist: Set<string> = new Set();
  private domainWhitelist: Set<string> = new Set();
  private similarityChecker = new DomainSimilarityChecker();
  
  async checkDomain(url: string): Promise<DomainRisk> {
    const domain = new URL(url).hostname;
    
    // 1. 白名单检查
    if (this.domainWhitelist.has(domain)) {
      return { safe: true, score: 100 };
    }
    
    // 2. 黑名单检查
    if (this.domainBlacklist.has(domain)) {
      return { safe: false, score: 0, reason: '已知钓鱼网站' };
    }
    
    // 3. 相似域名检测（typosquatting）
    const similarTo = this.similarityChecker.findSimilar(domain, [...this.domainWhitelist]);
    if (similarTo.similarity > 0.8) {
      return {
        safe: false,
        score: 20,
        reason: `疑似仿冒 ${similarTo.domain}，相似度 ${(similarTo.similarity * 100).toFixed(1)}%`
      };
    }
    
    // 4. 域名年龄检查
    const age = await this.getDomainAge(domain);
    if (age < 7) { // 7天内新注册
      return { safe: false, score: 40, reason: '域名新注册，请谨慎' };
    }
    
    // 5. 证书检查
    const certValid = await this.checkCertificate(url);
    if (!certValid) {
      return { safe: false, score: 30, reason: '证书异常' };
    }
    
    return { safe: true, score: 80 };
  }
  
  // 域名相似度检测
  private checkSimilarity(domain1: string, domain2: string): number {
    // 使用 Levenshtein 距离计算相似度
    const distance = levenshtein(domain1, domain2);
    const maxLen = Math.max(domain1.length, domain2.length);
    return 1 - distance / maxLen;
  }
}

// 合约风险检测
class ContractRiskScanner {
  private provider: ethers.Provider;
  private bytecodeAnalyzer: BytecodeAnalyzer;
  
  async scanContract(address: string): Promise<ContractRisk> {
    const code = await this.provider.getCode(address);
    
    if (code === '0x') {
      return { risk: 'none', reason: 'EOA 地址' };
    }
    
    const risks: RiskItem[] = [];
    
    // 1. 检查是否为代理合约
    const implementation = await this.detectImplementation(address);
    if (implementation) {
      risks.push({
        level: 'medium',
        type: 'proxy',
        description: '代理合约，实际逻辑可能在别处'
      });
    }
    
    // 2. 字节码分析
    const analysis = this.bytecodeAnalyzer.analyze(code);
    
    if (analysis.hasSelfdestruct) {
      risks.push({
        level: 'high',
        type: 'selfdestruct',
        description: '包含自毁函数，合约可被删除'
      });
    }
    
    if (analysis.hasUncheckedCall) {
      risks.push({
        level: 'high',
        type: 'reentrancy',
        description: '可能存在重入攻击风险'
      });
    }
    
    if (analysis.hasDelegateCall) {
      risks.push({
        level: 'medium',
        type: 'delegatecall',
        description: '使用 delegatecall，权限可能被转移'
      });
    }
    
    // 3. 蜜罐检测
    const isHoneypot = await this.checkHoneypot(address);
    if (isHoneypot) {
      risks.push({
        level: 'dangerous',
        type: 'honeypot',
        description: '疑似蜜罐合约，可能无法卖出'
      });
    }
    
    // 4. 外部数据库查询
    const external = await this.queryExternalDB(address);
    if (external.flagged) {
      risks.push({
        level: 'dangerous',
        type: 'blacklisted',
        description: `被标记: ${external.reason}`
      });
    }
    
    return {
      risk: this.calculateOverallRisk(risks),
      risks,
      verified: external.verified
    };
  }
  
  // 蜜罐检测
  private async checkHoneypot(address: string): Promise<boolean> {
    // 模拟买入卖出检测
    try {
      // 检查 transfer 函数是否有限制
      const sellSimulation = await this.simulateSell(address);
      return sellSimulation.reverted;
    } catch {
      return false;
    }
  }
  
  // 代理合约检测
  private async detectImplementation(address: string): Promise<string | null> {
    // EIP-1967 代理槽位
    const EIP1967_SLOT = '0x360894a13ba1a3210667c828492db98dca3e2076cc3735a920a3ca505d382bbc';
    
    const implementation = await this.provider.getStorage(address, EIP1967_SLOT);

class SignatureInterceptor {

    'eth_sign',           // 盲签，可签任意数据
    'personal_sign',      // 相对安全，但仍需检查
  ];
  
  async intercept(method: string, params: any[]): Promise<InterceptResult> {
    switch (method) {
      case 'eth_sign':
        return this.interceptEthSign(params);
      case 'personal_sign':
        return this.interceptPersonalSign(params);
      case 'eth_signTypedData_v4':
        return this.interceptTypedData(params);
      case 'eth_sendTransaction':
        return this.interceptTransaction(params);
      default:
        return { allowed: true };
    }
  }
  
  private interceptEthSign(params: any[]): InterceptResult {
    const [address, data] = params;
    
    // eth_sign 可以签署任意数据，包括交易哈希
    return {
      allowed: false,
      reason: 'eth_sign 已被弃用，它允许签署任意数据（包括交易），请使用 personal_sign 或 signTypedData',
      alternatives: ['personal_sign', 'eth_signTypedData_v4']
    };
  }
  
  private async interceptTypedData(params: any[]): Promise<InterceptResult> {
    const [address, jsonData] = params;
    const data = JSON.parse(jsonData);
    
    // 检查是否为 Permit
    if (this.isPermit(data)) {
      const risk = await this.assessPermitRisk(data);
      if (risk.level === 'high') {
        return {
          allowed: false,
          reason: `Permit 授权风险: ${risk.reason}`,
          details: risk
        };
      }
    }
    
    // 检查是否为 setApprovalForAll
    if (this.isSetApprovalForAll(data)) {
      return {
        allowed: false,
        reason: 'setApprovalForAll 会授予对某个 NFT 系列的所有代币的操作权限',
        warning: '请确认您信任该合约'
      };
    }
    
    return { allowed: true };
  }
  
  private isPermit(data: any): boolean {
    const permitTypes = ['Permit', 'PermitSingle', 'PermitBatch'];
    return permitTypes.includes(data.primaryType);
  }
  
  private isSetApprovalForAll(data: any): boolean {
    // 检查 EIP-712 数据中的 method
    return data.message?.function?.includes('setApprovalForAll');
  }
}
```

```typescript
      ],
      description: 'Solidity <0.8.0 版本中整数运算可能溢出'
    });

        const match = code.match(regex);
        if (match) {
          matches.push({
            id,
            name: pattern.name,
            severity: pattern.severity,
            line: this.getLineNumber(code, match.index!),
            description: pattern.description
          });
        }
      }
    }
    
    return matches;
  }
```

```typescript
class TradingStrategyPlanner {
  async planSwap(intent: SwapIntent): Promise<ExecutionPlan> {
    // 1. 获取所有可能的交易路径
    const routes = await this.findRoutes(
      intent.inputToken,
      intent.outputToken,
      intent.amount
    );
    
    // 2. 根据用户偏好排序
    const scoredRoutes = routes.map(route => ({
      ...route,
      score: this.scoreRoute(route, intent.preferences)
    }));
    
    scoredRoutes.sort((a, b) => b.score - a.score);
    
    // 3. 选择最佳路径
    const bestRoute = scoredRoutes[0];
    
    // 4. 生成执行计划
    return {
      steps: [
        {
          type: 'approve',
          token: intent.inputToken,
          spender: bestRoute.dex,
          amount: intent.amount
        },
        {
          type: 'swap',
          dex: bestRoute.dex,
          path: bestRoute.path,
          amountIn: intent.amount,
          minAmountOut: this.calculateMinOut(bestRoute.expectedOut, intent.maxSlippage)
        }
      ],
      expectedOutcome: {
        outputAmount: bestRoute.expectedOut,
        gasCost: bestRoute.estimatedGas,
        priceImpact: bestRoute.priceImpact
      },
      alternatives: scoredRoutes.slice(1, 3)
    };
  }
  
  private scoreRoute(route: Route, preferences: Preferences): number {
    let score = 0;
    
    // 输出金额权重
    score += route.expectedOut * preferences.outputWeight;
    
    // Gas 成本权重（负向）
    score -= route.estimatedGas * preferences.gasWeight;
    
    // 滑点惩罚
    if (route.priceImpact > preferences.maxSlippage) {
      score -= 1000;
    }
    
    // DEX 偏好
    if (route.dex === preferences.preferDex) {
      score += 10;
    }
    
    return score;
  }
}

class SecureTransactionExecutor {
  private policy: SecurityPolicy;
  private wallet: Wallet;
  
  constructor(policy: SecurityPolicy) {
    this.policy = policy;
  }
  
  async execute(plan: ExecutionPlan): Promise<ExecutionResult> {
    // 1. 安全检查
    const securityCheck = await this.securityCheck(plan);
    if (!securityCheck.passed) {
      return {
        success: false,
        error: `安全检查失败: ${securityCheck.reason}`
      };
    }
    
    // 2. 价值检查
    const valueCheck = this.checkValue(plan);
    if (valueCheck.requiresConfirmation) {
      const confirmed = await this.requestUserConfirmation(plan, valueCheck);
      if (!confirmed) {
        return { success: false, error: '用户取消' };
      }
    }
    
    // 3. 模拟执行
    const simulation = await this.simulate(plan);
    if (!simulation.success) {
      return {
        success: false,
        error: `模拟执行失败: ${simulation.error}`
      };
    }
    
    // 4. 实际执行
    const results = [];
    for (const step of plan.steps) {
      try {
        const result = await this.executeStep(step);
        results.push(result);
        
        // 每步后检查状态
        if (!await this.verifyStep(result)) {
          throw new Error('步骤验证失败');
        }
      } catch (error) {
        // 回滚已执行的步骤
        await this.rollback(results);
        throw error;
      }
    }
    
    return { success: true, results };
  }
  
  private async securityCheck(plan: ExecutionPlan): Promise<SecurityCheck> {
    // 检查合约风险
    for (const step of plan.steps) {
      if (step.spender) {
        const risk = await this.checkContractRisk(step.spender);
        if (risk.level === 'dangerous') {
          return { passed: false, reason: `风险合约: ${risk.reason}` };
        }
      }
    }
    
    // 检查金额限制
    const totalValue = await this.calculateTotalValue(plan);
    if (totalValue > this.policy.maxTransactionValue) {
      return { passed: false, reason: '超出单笔交易限额' };
    }
    
    // 检查频率限制
    if (await this.isRateLimitExceeded()) {
      return { passed: false, reason: '交易频率超限' };
    }
    
    return { passed: true };
  }
  
  private checkValue(plan: ExecutionPlan): ValueCheck {
    const totalValue = plan.expectedOutcome.outputAmount +
                       plan.expectedOutcome.gasCost;
    
    return {
      requiresConfirmation: totalValue > this.policy.confirmationThreshold,
      estimatedValue: totalValue,
      riskLevel: totalValue > this.policy.highRiskThreshold ? 'high' : 'medium'
    };
  }
  
  private async simulate(plan: ExecutionPlan): Promise<SimulationResult> {
    // 使用 tenderly 或自建模拟节点
    const simulation = await this.simulator.simulate({
      from: this.wallet.address,
      steps: plan.steps
    });
    
    return {
      success: simulation.status,
      gasUsed: simulation.gasUsed,
      outputAmount: simulation.outputAmount,
      error: simulation.error
    };
  }
}
```

### 常见问题和解决方案

1. **意图理解错误**：AI 误解用户意图导致错误交易
   - 解决：低置信度时要求澄清 + 计划展示确认

2. **价格滑点**：执行时价格变化导致损失
   - 解决：设置最大滑点 + 模拟执行验证

3. **Gas 波动**：预估 Gas 与实际差异大
   - 解决：动态 Gas 策略 + 用户设置上限

4. **合约风险**：新合约可能有漏洞
   - 解决：只使用白名单 DEX + 合约审计检查
