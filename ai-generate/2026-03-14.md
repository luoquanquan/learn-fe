## 1. React Hooks 原理与使用场景详解【前端】

**考点**: React / Hooks / useState / useEffect / 函数组件

**答案要点**:

### 核心 Hooks

| Hook | 作用 | 使用场景 |
|------|------|----------|
| `useState` | 状态管理 | 组件内部状态 |
| `useEffect` | 副作用处理 | 数据获取、订阅、手动DOM操作 |
| `useContext` | 上下文消费 | 跨组件传递数据 |
| `useRef` | 引用保存 | 访问DOM、保存可变值 |
| `useMemo` | 记忆化计算 | 复杂计算缓存 |
| `useCallback` | 记忆化函数 | 子组件性能优化 |

### useState 原理

```javascript
function Counter() {
  const [count, setCount] = useState(0);
  
  return (
    <button onClick={() => setCount(count + 1)}>
      Count: {count}
    </button>
  );
}
```

**原理**：React 通过链表保存 Hooks 状态，按调用顺序匹配。

### useEffect 使用

```javascript
useEffect(() => {
  // 组件挂载时执行
  const subscription = subscribe();
  
  return () => {
    // 组件卸载时清理
    subscription.unsubscribe();
  };
}, [dependency]); // dependency 变化时重新执行
```

### 注意事项

1. **不要在循环中调用 Hooks**：必须保证调用顺序一致
2. **依赖数组要完整**：避免闭包问题
3. **useEffect 清理**：防止内存泄漏

## 2. CSR、SSR、SSG、ISR 渲染模式对比【前端】

**考点**: 渲染模式 / 性能优化 / Next.js / 应用场景

**答案要点**:

### 四种渲染模式对比

| 模式 | 全称 | 渲染时机 | 优点 | 缺点 | 适用场景 |
|------|------|----------|------|------|----------|
| **CSR** | Client-Side Rendering | 浏览器运行 JS 渲染 | 交互好、服务器压力小 | 首屏慢、SEO差 | 后台管理系统 |
| **SSR** | Server-Side Rendering | 每次请求服务器渲染 | 首屏快、SEO好 | 服务器压力大 | 内容密集型网站 |
| **SSG** | Static Site Generation | 构建时预渲染 | 性能最好、CDN友好 | 数据更新需重建 | 博客、文档 |
| **ISR** | Incremental Static Regeneration | 构建时+运行时增量更新 | 兼顾性能和实时性 | 配置复杂 | 电商、新闻 |

### Next.js 实现

```javascript
// CSR - 默认
function Page() {
  const [data, setData] = useState();
  useEffect(() => {
    fetchData().then(setData);
  }, []);
  return <div>{data}</div>;
}

// SSR
export async function getServerSideProps() {
  const data = await fetchData();
  return { props: { data } };
}

// SSG
export async function getStaticProps() {
  const data = await fetchData();
  return { props: { data } };
}

// ISR
export async function getStaticProps() {
  const data = await fetchData();
  return { 
    props: { data },
    revalidate: 60 // 60秒后重新生成
  };
}
```

## 3. 以太坊账户模型与地址生成原理【Web3】

**考点**: 以太坊 / 账户模型 / 公私钥 / 地址生成

**答案要点**:

### 账户类型

| 类型 | 特点 | 用途 |
|------|------|------|
| **EOA** | 外部账户，由私钥控制 | 用户钱包 |
| **合约账户** | 由代码控制，有存储 | 智能合约 |

### 地址生成流程

```
私钥 (64位十六进制)
    ↓
ECC 椭圆曲线算法 (secp256k1)
    ↓
公钥 (128位，去掉04前缀)
    ↓
Keccak-256 哈希
    ↓
取后20字节 (40位十六进制)
    ↓
以太坊地址 (0x + 40位)
```

### 代码实现

```javascript
const ethers = require('ethers');

// 生成随机私钥
const wallet = ethers.Wallet.createRandom();
console.log('Private Key:', wallet.privateKey);
console.log('Address:', wallet.address);

// 从私钥恢复
const wallet2 = new ethers.Wallet(privateKey);
```

### 地址校验

```javascript
// EIP-55 校验地址
function isValidAddress(address) {
  return ethers.utils.isAddress(address);
}

// 转为校验和地址
const checksumAddress = ethers.utils.getAddress(address);
```

## 5. Gas 机制与 Gas 优化策略【Web3】

**考点**: Gas / 交易费用 / 优化策略 / EIP-1559

**答案要点**:

### Gas 机制

```
交易费用 = Gas Used × Gas Price
```

**EIP-1559 费用结构**：
- **Base Fee**: 基础费用，由网络自动计算，销毁
- **Priority Fee**: 优先费用（小费），给矿工
- **Max Fee**: 用户愿意支付的最高费用

### Gas 优化技巧

```solidity
// 1. 使用 calldata 代替 memory
function process(uint[] calldata data) external {
  // calldata 更省 gas
}

// 2. 批量操作
function batchTransfer(address[] calldata to, uint[] calldata amounts) external {
  for (uint i = 0; i < to.length; i++) {
    transfer(to[i], amounts[i]);
  }
}

// 3. 使用短路求优
if (a && b) { } // a 为 false 时不会检查 b

// 4. 避免存储重复计算
uint256 public constant VALUE = 100; // 常量不占用存储
```

### Gas Limit 设置

| 操作 | Gas 估算 |
|------|----------|
| ETH 转账 | 21,000 |
| ERC-20 转账 | ~50,000 |
| 合约部署 | 根据大小 |

## 8. Transformer 架构与注意力机制【AI】

**考点**: Transformer / Self-Attention / BERT / GPT

**答案要点**:

### Transformer 架构

```
Input → Embedding + Positional Encoding
    ↓
Encoder (Self-Attention + FFN)
    ↓
Decoder (Masked Self-Attention + Cross-Attention + FFN)
    ↓
Output
```

### Self-Attention 原理

```
Query × Key^T → Softmax → × Value

注意力分数 = softmax(QK^T / √d_k)
输出 = Attention × V
```

### 模型对比

| 模型 | 架构 | 特点 |
|------|------|------|
| **BERT** | Encoder-only | 双向理解，适合分类、NER |
| **GPT** | Decoder-only | 单向生成，适合文本生成 |
| **T5** | Encoder-Decoder | 统一框架，翻译、摘要 |

### 应用场景

- **BERT**: 情感分析、命名实体识别
- **GPT**: 对话生成、代码补全
- **T5**: 机器翻译、文本摘要

## 9. CSS 盒模型与布局基础【前端】

**考点**: CSS / 盒模型 / Flexbox / 布局

**答案要点**:

### 盒模型

```css
.box {
  content: 内容区;
  padding: 内边距;
  border: 边框;
  margin: 外边距;
}
```

**box-sizing**:
- `content-box`: 默认值，width/height 只包含 content
- `border-box`: width/height 包含 content + padding + border

```css
* {
  box-sizing: border-box; /* 推荐 */
}
```

### Flexbox 布局

```css
.container {
  display: flex;
  justify-content: center; /* 主轴对齐 */
  align-items: center;     /* 交叉轴对齐 */
  gap: 20px;               /* 间距 */
}

.item {
  flex: 1;    /* 占据剩余空间 */
  flex: 0 0 200px; /* 不伸缩，固定200px */
}
```

### Grid 布局

```css
.grid {
  display: grid;
  grid-template-columns: repeat(3, 1fr);
  gap: 20px;
}
```
