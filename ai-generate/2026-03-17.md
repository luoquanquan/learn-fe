## 1. React forwardRef 作用是什么？如何使用？【前端】

**考点**: React / ref / 组件通信 / 高阶组件

**答案要点**:

### 核心作用

`forwardRef` 是 React 提供的高阶函数，让**函数组件可以接收并转发 ref 到子组件**。解决函数组件无法直接接收 ref 的限制。

### 使用场景

1. **操作子组件 DOM**：聚焦输入框、控制滚动
2. **封装第三方组件**：保持 ref 能力
3. **高阶组件**：传递 ref 给被包裹组件

### 代码示例

```jsx
import React, { forwardRef, useRef } from 'react';

// 使用 forwardRef 包装组件
const Input = forwardRef((props, ref) => {
  return <input type="text" ref={ref} {...props} />;
});

// 父组件使用
function Parent() {
  const inputRef = useRef(null);
  
  const handleFocus = () => {
    // 直接操作子组件的 DOM
    inputRef.current.focus();
  };
  
  return (
    <div>
      <Input ref={inputRef} placeholder="请输入" />
      <button onClick={handleFocus}>聚焦输入框</button>
    </div>
  );
}
```

### 配合 useImperativeHandle

```jsx
const FancyInput = forwardRef((props, ref) => {
  const inputRef = useRef();
  
  // 暴露特定方法给父组件
  useImperativeHandle(ref, () => ({
    focus: () => inputRef.current.focus(),
    clear: () => inputRef.current.value = '',
    getValue: () => inputRef.current.value,
  }));
  
  return <input ref={inputRef} {...props} />;
});

// 父组件只使用暴露的方法
function Parent() {
  const fancyRef = useRef();
  
  return (
    <>
      <FancyInput ref={fancyRef} />
      <button onClick={() => fancyRef.current.clear()}>清空</button>
    </>
  );
}
```

### 注意事项

1. **不要在函数组件上使用 ref**：函数组件没有实例，ref 无法获取
2. **forwardRef 只传递一个 ref**：多个 ref 需要合并或使用其他方案
3. **与 TypeScript 配合**：需要定义 ref 类型

```typescript
interface InputProps {
  value: string;
  onChange: (value: string) => void;
}

// 定义 ref 暴露的方法
interface InputRef {
  focus: () => void;
  clear: () => void;
}

const Input = forwardRef<InputRef, InputProps>((props, ref) => {
  // ...
});
```

## 2. Webpack Tree Shaking 什么情况下会失效？如何优化？【前端】

**考点**: Webpack / Tree Shaking / 代码优化 / 模块系统

**答案要点**:

### Tree Shaking 原理

Tree Shaking 基于 ES Module 的**静态分析**，在编译时识别未使用的代码并删除，减少打包体积。

### 失效场景

| 场景 | 原因 | 解决方案 |
|------|------|----------|
| **CommonJS 模块** | `require()` 动态导入，无法静态分析 | 使用 ES Module `import/export` |
| **副作用代码** | 模块修改全局变量、污染原型 | 配置 `sideEffects` |
| **Babel 转译问题** | `@babel/preset-env` 可能转为 CommonJS | 配置 `@babel/preset-env` 的 `modules: false` |
| **动态导入** | `import()` 运行时确定 | 无法 Tree Shaking，需手动优化 |
| **未使用的类方法** | 类方法无法单独 Tree Shaking | 使用函数替代，或配置优化 |

### 配置优化

```javascript
// package.json - 声明副作用
{
  "name": "my-app",
  "sideEffects": [
    "*.css",      // CSS 有副作用
    "*.scss",
    "./src/polyfill.js"  //  polyfill 有副作用
  ]
}

// 无副作用可以设为 false
{
  "sideEffects": false
}
```

```javascript
// webpack.config.js
module.exports = {
  mode: 'production',  // 生产模式自动启用 Tree Shaking
  
  optimization: {
    // 标记未使用的导出
    usedExports: true,
    
    // 识别 sideEffects 字段
    sideEffects: true,
    
    // 模块合并优化
    concatenateModules: true,
    
    // 代码分割
    splitChunks: {
      chunks: 'all',
      cacheGroups: {
        vendor: {
          test: /[\\/]node_modules[\\/]/,
          name: 'vendors',
          chunks: 'all'
        }
      }
    }
  }
};
```

```javascript
// babel.config.js - 防止转译为 CommonJS
module.exports = {
  presets: [
    ['@babel/preset-env', { 
      modules: false  // 保持 ES Module
    }]
  ]
};
```

**答案要点**:

### 交易结构

```javascript

  nonce: 0,                    // 交易序号
  gasPrice: 20000000000,       // Gas 价格 (wei)
  gasLimit: 21000,             // Gas 限制

  value: 1000000000000000000,  // 转账金额 (1 ETH = 10^18 wei)
  data: '0x',                  // 合约调用数据
  chainId: 1                   // 网络 ID (1=主网)
```

### 签名流程

```

└────────┬────────┘

┌─────────────────┐
│  2. RLP 编码    │  ← 递归长度前缀编码
└────────┬────────┘
         ▼
┌─────────────────┐
│  3. Keccak-256  │  ← 哈希算法
│     哈希        │
└────────┬────────┘
         ▼
┌─────────────────┐
│  4. ECDSA 签名  │  ← 使用私钥签名
│                 │
│   r = x坐标     │
│   s = y坐标     │
│   v = 恢复标识  │
└────────┬────────┘
         ▼
┌─────────────────┐
│  5. 序列化签名   │
│   发送交易      │
└─────────────────┘
```

## 4. 智能合约重入攻击原理与防护【Web3】

**考点**: 智能合约安全 / 重入攻击 / Reentrancy / 最佳实践

**答案要点**:

### 攻击原理

重入攻击利用 Solidity 的 `call` 可以触发目标合约的 fallback 函数，形成递归调用。

```solidity
// 漏洞合约：先转账，后更新余额
contract VulnerableBank {
  mapping(address => uint) public balances;
  
  function deposit() public payable {
    balances[msg.sender] += msg.value;
  }
  
  function withdraw() public {
    uint amount = balances[msg.sender];
    
    // ⚠️ 漏洞：先转账，后更新余额
    (bool success, ) = msg.sender.call{value: amount}("");
    require(success, "Transfer failed");
    
    balances[msg.sender] = 0;  // 更新太晚了！
  }
}

// 攻击合约
contract Attacker {
  VulnerableBank public bank;
  
  constructor(address _bank) {
    bank = VulnerableBank(_bank);
  }
  
  // 先存入一些 ETH
  function attack() external payable {
    bank.deposit{value: msg.value}();
    bank.withdraw();
  }
  
  // fallback 函数：收到转账时再次调用 withdraw
  receive() external payable {
    if (address(bank).balance >= 1 ether) {
      bank.withdraw();  // 重入！
    }
  }
}
```

### 攻击流程

```
1. Attacker 存入 1 ETH
2. Attacker 调用 withdraw()
3. VulnerableBank 发送 1 ETH 给 Attacker
4. 触发 Attacker 的 receive()
5. receive() 再次调用 withdraw()
6. 重复步骤 3-5，直到银行 ETH 耗尽
7. balances[msg.sender] = 0 才执行
```

### 防护措施

#### 方案 1: Checks-Effects-Interactions 模式（推荐）

```solidity
contract SecureBank {
  mapping(address => uint) public balances;
  
  function withdraw() public {
    uint amount = balances[msg.sender];
    
    // 1. Checks：检查条件
    require(amount > 0, "No balance");
    
    // 2. Effects：先更新状态
    balances[msg.sender] = 0;
    
    // 3. Interactions：最后与外部交互
    (bool success, ) = msg.sender.call{value: amount}("");
    require(success, "Transfer failed");
  }
}
```

#### 方案 2: ReentrancyGuard（OpenZeppelin）

```solidity
import "@openzeppelin/contracts/security/ReentrancyGuard.sol";

contract SecureBank is ReentrancyGuard {
  mapping(address => uint) public balances;
  
  // nonReentrant 修饰器防止重入
  function withdraw() public nonReentrant {
    uint amount = balances[msg.sender];
    require(amount > 0, "No balance");
    
    (bool success, ) = msg.sender.call{value: amount}("");
    require(success, "Transfer failed");
    
    balances[msg.sender] = 0;
  }
}

// OpenZeppelin 实现
abstract contract ReentrancyGuard {
  uint256 private constant _NOT_ENTERED = 1;
  uint256 private constant _ENTERED = 2;
  uint256 private _status;
  
  modifier nonReentrant() {
    require(_status != _ENTERED, "ReentrancyGuard: reentrant call");
    _status = _ENTERED;
    _;
    _status = _NOT_ENTERED;
  }
}
```

#### 方案 3: 使用 transfer/send（不推荐新使用）

```solidity
function withdraw() public {
  uint amount = balances[msg.sender];
  require(amount > 0, "No balance");
  
  // transfer 限制 gas 为 2300，无法触发复杂逻辑
  payable(msg.sender).transfer(amount);
  
  balances[msg.sender] = 0;
}
```

### 真实案例

**The DAO 攻击（2016）**：
- 被盗 360 万 ETH（当时价值约 6000 万美元）
- 导致以太坊硬分叉，产生 ETH 和 ETC

## 5. ERC-20、ERC-721、ERC-1155 核心区别与应用场景【Web3】

**考点**: ERC标准 / 代币类型 / NFT / 多代币标准

**答案要点**:

### 标准对比

| 特性 | ERC-20 | ERC-721 | ERC-1155 |
|------|--------|---------|----------|
| **同质化** | ✅ 可互换 | ❌ 唯一 | 混合 |
| **可分割** | ✅ 可拆分 | ❌ 不可分割 | 可选 |
| **代表** | USDT、UNI | CryptoPunks | 游戏道具 |
| **转账函数** | `transfer(to, amount)` | `transferFrom(from, to, tokenId)` | `safeTransferFrom(from, to, id, amount, data)` |
| **授权方式** | `approve(spender, amount)` | `approve(spender, tokenId)` | `setApprovalForAll(operator, approved)` |

### ERC-20（同质化代币）

```solidity
interface IERC20 {
  function totalSupply() external view returns (uint256);
  function balanceOf(address account) external view returns (uint256);
  function transfer(address to, uint256 amount) external returns (bool);
  function allowance(address owner, address spender) external view returns (uint256);
  function approve(address spender, uint256 amount) external returns (bool);
  function transferFrom(address from, address to, uint256 amount) external returns (bool);
  
  event Transfer(address indexed from, address indexed to, uint256 value);
  event Approval(address indexed owner, address indexed spender, uint256 value);
}
```

**使用场景**：货币、积分、投票权、流动性代币

### ERC-721（非同质化代币，NFT）

```solidity
interface IERC721 {
  function balanceOf(address owner) external view returns (uint256);
  function ownerOf(uint256 tokenId) external view returns (address);
  function safeTransferFrom(address from, address to, uint256 tokenId) external;
  function transferFrom(address from, address to, uint256 tokenId) external;
  function approve(address to, uint256 tokenId) external;
  function getApproved(uint256 tokenId) external view returns (address);
  function setApprovalForAll(address operator, bool approved) external;
  function isApprovedForAll(address owner, address operator) external view returns (bool);
  
  event Transfer(address indexed from, address indexed to, uint256 indexed tokenId);
  event Approval(address indexed owner, address indexed approved, uint256 indexed tokenId);
}
```

**使用场景**：数字艺术、收藏品、游戏角色、域名、门票

### ERC-1155（多代币标准）

```solidity
interface IERC1155 {
  // 查询余额（支持批量）
  function balanceOf(address account, uint256 id) external view returns (uint256);
  function balanceOfBatch(address[] calldata accounts, uint256[] calldata ids) external view returns (uint256[] memory);
  
  // 批量转账
  function safeBatchTransferFrom(
    address from,
    address to,
    uint256[] calldata ids,
    uint256[] calldata amounts,
    bytes calldata data
  ) external;
  
  // 全局授权
  function setApprovalForAll(address operator, bool approved) external;
  function isApprovedForAll(address account, address operator) external view returns (bool);
}
```

**优势**：
- 一个合约管理多种代币
- 批量转账节省 Gas
- 同时支持同质化和非同质化

**使用场景**：游戏装备（金币+武器+皮肤）、门票系统、优惠券

### Gas 成本对比

| 操作 | ERC-20 | ERC-721 | ERC-1155 |
|------|--------|---------|----------|
| 转账 | ~21,000 | ~50,000 | ~30,000 |
| 批量转账（10个）| 10 × 21,000 | 10 × 50,000 | ~35,000 |

## 6. MetaMask 钱包注入原理与 EIP-1193 标准【Web3】

**考点**: 钱包连接 / EIP-1193 / Provider / 浏览器扩展

**答案要点**:

### 注入原理

MetaMask 浏览器扩展通过 **内容脚本（Content Script）** 将 `ethereum` 对象注入到网页的 `window` 中：

```
网页 (DApp)
    ↑
window.ethereum  ← 注入
    ↑
MetaMask 内容脚本
    ↑
MetaMask 扩展后台
    ↑
用户私钥（加密存储）
```

### EIP-1193 标准

EIP-1193 定义了以太坊 Provider 的标准接口，统一钱包连接方式。

```typescript
interface EthereumProvider {
  // 请求方法
  request(args: { method: string; params?: unknown[] }): Promise<unknown>;
  
  // 事件监听
  on(event: 'accountsChanged', handler: (accounts: string[]) => void): void;
  on(event: 'chainChanged', handler: (chainId: string) => void): void;
  on(event: 'connect', handler: (info: { chainId: string }) => void): void;
  on(event: 'disconnect', handler: (error: Error) => void): void;
  
  // 移除监听
  removeListener(event: string, handler: Function): void;
}
```

### 代码示例

```javascript
// 1. 检测钱包
if (typeof window.ethereum !== 'undefined') {
  console.log('MetaMask is installed!');
}

// 2. 请求账户连接
async function connectWallet() {
  try {
    const accounts = await window.ethereum.request({
      method: 'eth_requestAccounts'
    });
    
    console.log('Connected:', accounts[0]);
    return accounts[0];
  } catch (error) {
    if (error.code === 4001) {
      console.log('User rejected the request');
    }
  }
}

// 3. 监听账户变化
window.ethereum.on('accountsChanged', (accounts) => {
  if (accounts.length === 0) {
    console.log('User disconnected');
  } else {
    console.log('Account changed:', accounts[0]);
  }
});

// 4. 监听链变化
window.ethereum.on('chainChanged', (chainId) => {
  // 链变化时刷新页面
  window.location.reload();
});

// 5. 获取当前链 ID
const chainId = await window.ethereum.request({
  method: 'eth_chainId'
});
console.log('Current chain:', parseInt(chainId, 16));  // 0x1 = 1 (主网)
```

### 多钱包兼容性（EIP-6963）

```typescript
// 监听钱包广播
window.addEventListener('eip6963:announceProvider', (event) => {
  const { info, provider } = event.detail;

  const selectedProvider = provider;

```

### 安全注意事项

1. **不要信任前端传来的地址**：后端必须验证签名
2. **监听网络变化**：防止用户切换到错误网络
3. **处理拒绝情况**：用户可能拒绝连接或签名
4. **不要硬编码 RPC**：使用用户钱包的网络

## 7. RAG 检索增强生成原理与实现【AI】

**考点**: RAG / LLM / 向量数据库 / 检索系统

**答案要点**:

### 什么是 RAG

**RAG（Retrieval-Augmented Generation）**：结合信息检索和文本生成的技术，让大模型基于外部知识回答问题，减少幻觉。

### 系统架构

```
┌─────────────────────────────────────────────────────────────┐
│                         用户查询                              │
└────────────────────────┬────────────────────────────────────┘
                         ▼
┌─────────────────────────────────────────────────────────────┐
│                    查询向量化 (Embedding)                      │
│              使用模型：text-embedding-ada-002                 │
└────────────────────────┬────────────────────────────────────┘
                         ▼
┌─────────────────────────────────────────────────────────────┐
│                    向量相似度检索                              │
│              向量数据库：Pinecone / Milvus / Weaviate          │
└────────────────────────┬────────────────────────────────────┘
                         ▼
┌─────────────────────────────────────────────────────────────┐
│                    构建增强 Prompt                             │
│         "基于以下内容回答问题：[检索到的文档]\n\n问题：[查询]"    │
└────────────────────────┬────────────────────────────────────┘
                         ▼
┌─────────────────────────────────────────────────────────────┐
│                      LLM 生成答案                             │
│              使用模型：GPT-4 / Claude / Llama                  │
└─────────────────────────────────────────────────────────────┘
```

### 核心组件

| 组件 | 作用 | 常用工具 |
|------|------|----------|
| **Embedding 模型** | 文本向量化 | OpenAI text-embedding-ada-002, BGE, M3E |
| **向量数据库** | 存储和检索向量 | Pinecone, Milvus, Weaviate, Chroma |
| **LLM** | 生成答案 | GPT-4, Claude, Llama 2 |
| **文档加载器** | 加载各种格式文档 | LangChain, LlamaIndex |

### 代码实现

```python
]
vectorstore.add_texts(docs)

# 5. 创建 RAG 链
llm = ChatOpenAI(model_name="gpt-4", temperature=0)
qa_chain = RetrievalQA.from_chain_type(
    llm=llm,
    chain_type="stuff",
    retriever=vectorstore.as_retriever(search_kwargs={"k": 3})
)

# 6. 查询
query = "什么是 Gas Fee？"
result = qa_chain.run(query)
print(result)
```

### 优化技巧

1. **文档分块**：长文档切分成小块，提高检索精度
2. **重排序（Re-rank）**：初筛后用更精确的模型重排序
3. **混合检索**：向量检索 + 关键词检索结合
4. **上下文压缩**：只保留最相关的句子，节省 Token

## 9. Babel 工作流程与 AST 转换原理【前端】

**考点**: Babel / AST / 代码转换 / JavaScript 编译

**答案要点**:

### Babel 是什么

Babel 是 JavaScript 编译器，将新版 ES 代码转换为兼容旧版浏览器的代码。

### 三阶段工作流程

```
┌──────────┐    ┌──────────┐    ┌──────────┐
│  源码     │ → │   AST    │ → │ 目标代码  │
│ (ES2024) │    │ (抽象语法树)│    │  (ES5)   │
└──────────┘    └──────────┘    └──────────┘
     ↑              ↑               ↑
   Parse        Transform        Generate
  (解析)        (转换)           (生成)
```

### 1. Parse（解析）

将源码转换为 AST（抽象语法树）：

```javascript
// 源码
const add = (a, b) => a + b;

// AST 结构
{
  "type": "VariableDeclaration",
  "declarations": [{
    "type": "VariableDeclarator",
    "id": { "type": "Identifier", "name": "add" },
    "init": {
      "type": "ArrowFunctionExpression",
      "params": [
        { "type": "Identifier", "name": "a" },
        { "type": "Identifier", "name": "b" }
      ],
      "body": {
        "type": "BinaryExpression",
        "operator": "+",
        "left": { "type": "Identifier", "name": "a" },
        "right": { "type": "Identifier", "name": "b" }
      }
    }
  }]
}
```

### 2. Transform（转换）

遍历和修改 AST：

```javascript
// 箭头函数转普通函数
// 转换前
const add = (a, b) => a + b;

// 转换后
var add = function(a, b) {
  return a + b;
};
```

### 3. Generate（生成）

将 AST 转换回代码字符串。

### 配置示例

```javascript
// babel.config.js
module.exports = {
  presets: [
    ['@babel/preset-env', {
      targets: {
        browsers: ['> 1%', 'last 2 versions', 'not dead']
      },
      useBuiltIns: 'usage',
      corejs: 3
    }],
    '@babel/preset-react',
    '@babel/preset-typescript'
  ],
  plugins: [
    '@babel/plugin-proposal-class-properties',
    '@babel/plugin-proposal-decorators'
  ]
};
```

### Preset 与 Plugin 区别

| 类型 | 作用 | 示例 |
|------|------|------|
| **Preset** | 一组插件的集合 | `@babel/preset-env` |
| **Plugin** | 单一转换功能 | `@babel/plugin-transform-arrow-functions` |

## 10. AI Agent 自动化交易与风控系统设计【AI】

**考点**: AI Agent / 自动化交易 / 风险控制 / 策略引擎

**答案要点**:

### 系统架构

```
市场数据 → 策略引擎 → 风险评估 → 执行器 → 区块链
              ↓           ↓
         策略配置      风控规则
```

### 核心组件

```typescript

interface Strategy {
  evaluate(data: MarketData): Signal | null;
}

// 2. 风控引擎
interface RiskEngine {

// 3. 执行器
```

### DCA（定投）策略示例

```typescript
class DCAStrategy implements Strategy {
  private lastBuyTime: number = 0;
  private interval: number = 7 * 24 * 60 * 60 * 1000; // 7天
  private amount: number = 100; // 每次购买金额
  
  evaluate(data: MarketData): Signal | null {
    const now = Date.now();
    
    if (now - this.lastBuyTime >= this.interval) {
      this.lastBuyTime = now;
      return {
        action: 'BUY',
        token: 'ETH',
        amount: this.amount
      };
    }
    
    return null;
  }
}
```

### 风控规则

```typescript
class RiskEngine {
  private rules: RiskRule[] = [
    new MaxDailyLossRule(0.05),      // 日亏损不超过 5%
    new MaxPositionRule(0.3),         // 单币种仓位不超过 30%
    new CooldownRule(60 * 60 * 1000), // 交易间隔 1 小时
    new WhitelistRule()               // 只交易白名单代币
  ];
  
  assess(signal: Signal): RiskResult {
    for (const rule of this.rules) {
      const result = rule.check(signal);
      if (!result.passed) {
        return { approved: false, reason: result.reason };
      }
    }
    return { approved: true };
  }
}
```

### 人工确认机制

```typescript
class HumanInTheLoop {
  async requestApproval(signal: Signal): Promise<boolean> {
    // 发送通知到用户
    await notifyUser({
```

### 安全要点

1. **私钥分离**：使用 Session Key 或智能合约钱包
2. **限额控制**：设置单笔/日累计交易上限
3. **人工确认**：超过阈值的交易需人工确认
4. **监控告警**：异常行为实时告警
5. **熔断机制**：连续亏损时自动停止
