## 面试题设计原则

### 问题层级
1. **项目经验层**："你在 OKX 负责硬件钱包接入，能详细讲讲统一协议的设计思路吗？"
2. **技术原理层**："硬件钱包的 APDU 指令格式是怎样的？如何设计一个通用的抽象层？"
3. **方案对比层**："对比 Ledger 和 Keystone 的通信协议差异，你的统一层如何解决这些差异？"
4. **延伸拓展层**："如果让你设计一个新的硬件钱包标准，你会考虑哪些关键因素？"

### 答案结构要求
```
1. 项目背景（OKX 场景）
2. 技术方案设计
3. 核心代码示例
4. 遇到的问题与解决
5. 延伸思考（通用方法论）
```
